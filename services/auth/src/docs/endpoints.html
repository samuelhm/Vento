Guía de Integración: Servicio de Autenticación

Nota Importante para Front-end: Autenticación

  • Mecanismo: Cookies HTTP-Only. No es necesario manejar tokens manualmente en el código.
  • Configuración: El cliente (Axios/Fetch) DEBE incluir la opción withCredentials: true o credentials: 'include' en cada petición para que el navegador envíe la cookie.
  • XSS: La cookie no es accesible vía JavaScript por seguridad.
POST /signup Registro de nuevo usuario

Request Body (JSON)

{
  "name": "Juan",
  "lastNames": "Pérez",
  "email": "juan@example.com",
  "password": "mypassword123",
  "lat": 40.4167,
  "lng": -3.7033
}

Respuestas

CódigoDescripción
201 CreatedUsuario creado. Recibe cookie 'token' y objeto user.
409 ConflictEl email ya está registrado.
POST /signin Inicio de sesión

Request Body

{
  "email": "juan@example.com",
  "password": "mypassword123"
}

Respuestas

CódigoDescripción
200 OKLogin exitoso. Recibe cookie 'token'.
401 UnauthorizedCredenciales incorrectas.
GET /user/:id Ver perfil de usuario

* Si el ID del token coincide con el :id, se muestran datos privados (email, coords). Si no, vista pública.

URL Params

idUUID Requerido

Respuestas

200 OKRetorna objeto usuario filtrado.
404 Not FoundEl usuario no existe.
PUT /user/:id Actualizar datos de perfil

Seguridad: Solo permitido si el usuario autenticado es el dueño del :id.

Request Body (Opcionales)

{
  "name": "Nuevo Nombre",
  "lastNames": "Nuevos Apellidos",
  "lat": 41.3851,
  "lng": 2.1734
}
403 ForbiddenIntentas editar un perfil que no es tuyo.
POST /logout Cerrar sesión

Limpia la cookie del navegador inmediatamente.

DELETE /user/:id Eliminar cuenta

Seguridad: Requiere estar autenticado como el dueño de la cuenta.

200 OKCuenta borrada y cookie eliminada.
403 ForbiddenNo tienes permiso para borrar este usuario.
