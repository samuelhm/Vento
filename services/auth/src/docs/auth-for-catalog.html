Guía para usar el token del servicio Auth

Esta guía explica cómo validar la identidad del usuario en el servicio de Catalog utilizando la sesión iniciada en el servicio de Auth.

Flujo de Verificación:
Cliente (Browser) → Envía Cookie token automáticamente → Catalog Service (Valida JWT) → Acceso Concedido

1. Requisitos Previos

  • Ambos servicios deben compartir la misma JWT_SECRET en el archivo .env.
  • El servicio de Catalog debe tener instalados @fastify/jwt y @fastify/cookie.

2. Configuración en Catalog (index.js)

Registrar los plugins exactamente igual para que puedan leer la cookie enviada por el Auth Service:

fastify.register(import('@fastify/cookie'));
fastify.register(import('@fastify/jwt'), {
  secret: process.env.JWT_SECRET,
  cookie: {
    cookieName: 'token',
    signed: false
  }
});

3. ¿Cómo proteger una ruta?

Para verificar si el usuario está logueado antes de mostrar el catálogo o permitir compras, debe usar request.jwtVerify().

Opción A: Verificación Manual en el Handler

Útil si la ruta es "semi-privada" (muestra cosas distintas si estás logueado o no).

// catalog-controller.js
export const getProducts = async (request, reply) => {
  try {
    // Esto busca la cookie, la desencripta y mete los datos en request.user
    await request.jwtVerify();
    const userId = request.user.id;
    
    return { message: `Hola usuario ${userId}, aquí tienes tus ofertas personalizadas` };
  } catch (err) {
    // Si no hay token o es inválido, mostramos catálogo genérico
    return { message: "Aquí tienes el catálogo público" };
  }
};
Opción B: Verificación Obligatoria (PreHandler)

Ideal para acciones que requieren sí o sí estar autenticado (ej: añadir al carrito).

// catalog-routes.js
fastify.post('/cart/add', {
  preHandler: [async (request, reply) => {
    try {
      await request.jwtVerify();
    } catch (err) {
      reply.code(401).send({ error: 'Debes iniciar sesión para comprar' });
    }
  }],
  handler: catalogController.addToCart
});

4. Datos disponibles

Una vez que await request.jwtVerify() tiene éxito, tendrá acceso a:

  • request.user.id: El UUID del usuario.
  • request.user.email: El correo del usuario.

⚠️ IMPORTANTE: No necesitas consultar la base de datos de usuarios desde Catalog para saber quién es el usuario. Los datos básicos ya vienen dentro del token que viaja en la cookie.
